# 08 Delivery And Handoff

## Workstreams
- source collection
- rule normalization
- page generation and authority routing
- provider evidence and operator routing
- operator tools
- freshness and verification

## Milestone 1
- choose the first cluster
- publish first authority-backed local rule pages
- publish first verdict pages
- publish first provider-routing pages
- publish first operator tools

## Milestone 2
- add authority browse and trust surfaces
- deepen source freshness and noindex promotion controls
- expand to the second jurisdiction cluster

## Milestone 3
- multi-location workflows
- source change alerts
- stronger repeat-use operator utilities

## Acceptance criteria
- A new agent can implement from this folder without chat history
- No city page launches without local source coverage
- Official guidance and provider routing stay clearly separate
- The product resolves a next action, not just a rule summary
- Any future provider-sales workflow is explicitly out of scope for this repo

## Handoff order
1. Sourcing owner
2. Backend and registry owner
3. Page generation owner
4. Product ops owner
